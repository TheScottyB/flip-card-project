Getting Started

To use the flip card component in your project, include the CSS and JavaScript files: 


<link rel="stylesheet" href="dist/css/flip-card.min.css">
<script src="dist/js/flip-card.min.js"></script>

HTML Structure

Use the following HTML structure for creating a flip card: 


<div class="flip-card card-standard">
  <div class="flip-card-inner">
    <div class="flip-card-front">
      <!-- Front content -->
      <button class="flip-trigger">Flip Card</button>
    </div>
    <div class="flip-card-back">
      <!-- Back content -->
      <button class="flip-trigger">Return</button>
    </div>
  </div>
</div>

CSS Classes

  • flip-card - The main container for the card
  • card-standard, card-mini, card-tall - Size variants
  • flip-card-inner - The inner container that rotates
  • flip-card-front - The front side of the card
  • flip-card-back - The back side of the card
  • flip-trigger - Buttons that trigger the flip action

Accessibility Features

All flip card components are built with accessibility in mind and conform to WCAG 2.1 standards:

  • Keyboard Navigation - Cards can be flipped using Enter, Space, and Escape keys
  • Focus Management - Focus is properly trapped within the flipped card
  • ARIA Attributes - Proper ARIA roles, states, and properties are used
  • Screen Reader Support - Status changes are announced via ARIA live regions
  • Reduced Motion - Animation is disabled for users with reduced motion preferences
  • Color Contrast - All text meets WCAG AA color contrast requirements
  • Progressive Enhancement - Fallbacks for browsers that don't support 3D transforms

Accessibility Testing

The card components have been tested with:

  • VoiceOver on macOS and iOS
  • NVDA on Windows
  • Keyboard-only navigation
  • Automated testing using Axe
